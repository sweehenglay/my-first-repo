Deploy a live website

Here is your step-by-step guide to making my-first-repo live:

1. Enable GitHub Pages in Your Repository

  1. Open your browser and go to your repository.

  2. Look at the menu bar right below your repository name and click on the Settings icon (the gear icon on the far right).

  3. On the left-hand sidebar, scroll down to the Code and automation section and click on Pages.

2. Configure the Build and Deployment Source

  1. Under the Build and deployment section, look for Source. Ensure it is set to Deploy from a branch (this is usually the default).

  2. Under Branch, click the dropdown menu that currently says None and select main.

  3. Leave the folder dropdown right next to it set to / (root) (since your index.html is in the main folder).

  4. Click the Save button.

3. See Your Live Website!

  • GitHub will now take about 1 to 2 minutes to build and deploy your site.
  • Refresh the page after a minute. At the top of that same Pages settings screen, you will see a live link boxed in a green background that says: “Your site is live at…”

💡 Quick Tip: If you ever make changes to your file in VS Code and push them to GitHub using the workflows you learned in the last session, your live website will automatically update itself a minute later!
